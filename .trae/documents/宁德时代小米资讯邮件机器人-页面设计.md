# 页面设计（Desktop-first）

## 全局设计规范
- Layout：以 CSS Grid + Flexbox 混合为主。桌面端采用 12 列栅格（最大内容宽度 1120–1200px），卡片内部使用 Flex 垂直排布。
- Responsive：
  - ≥1024px：左右留白 + 2 列信息卡布局；消息列表为表格视图。
  - 640–1023px：信息卡改为单列；表格降级为列表视图。
  - <640px：信息卡与列表全部单列，减少列信息，保留标题/时间/链接。
- 设计令牌（建议值，可在 Tailwind 中落地）：
  - 背景色：#0B1220（深色）或 #F7F8FA（浅色，默认）
  - 主色：#2563EB（按钮/链接）
  - 成功：#16A34A；失败：#DC2626；警告：#F59E0B
  - 字体：系统字体栈；字号：12/14/16/20/24
  - 卡片：圆角 12px，边框 1px #E5E7EB，阴影轻量
  - 交互：按钮 hover 加深 8–12%，表格行 hover 轻背景

## 1) 任务状态页（首页）

### 1. Layout
- 页面框架：Header（固定高度）+ Main（内容容器）+ Footer（可选，最小化）。
- Main 采用 Grid：
  - 上半区：信息概览卡片（2 列：左=运行状态；右=增量与统计）。
  - 下半区：最新消息列表（占满宽度）。

### 2. Meta Information
- Title：宁德时代/小米资讯日报 - 任务状态
- Description：展示每日 8 点抓取与邮件发送结果、增量窗口与最新消息。
- Open Graph：
  - og:title 同 Title
  - og:description 同 Description
  - og:type = website

### 3. Page Structure
1. Header
2. 状态概览区（Cards）
3. 最新消息区（Table/List）

### 4. Sections & Components

#### 4.1 Header
- 左侧：产品名称“资讯日报机器人”。
- 右侧：关键固定信息（只读）：
  - “计划：每天 08:00”
  - “邮箱：{指定邮箱}”（如出于安全考虑可只显示部分掩码）

#### 4.2 状态概览区（两张卡片）
- 卡片 A：运行状态
  - 字段：当前状态（空闲/运行中/失败）
  - 字段：最近一次执行时间
  - 字段：最近一次结果（成功/失败 + 简短原因）
- 卡片 B：增量与去重统计
  - 字段：上次成功时间点（last_success_at）
  - 字段：本次时间窗口（window_start ~ window_end）
  - 字段：抓取条数（fetched_count）
  - 字段：去重命中（deduped_count）
  - 字段：最终邮件条目（output_count）
  - 字段：邮件发送时间与状态

展示规则：
- 成功：状态标签绿色；失败：红色；运行中：黄色/蓝色。
- 时间统一显示本地化格式（例如：YYYY-MM-DD HH:mm）。

#### 4.3 最新消息区
- 标题行：“本次新增消息（用于核对邮件）”。
- 桌面端表格列：
  - 公司（宁德时代/小米）
  - 标题（可换行，最多 2 行，溢出省略）
  - 来源
  - 发布时间
  - 链接（按钮/外链图标，点击新窗口打开）
- 移动端列表：
  - 第一行：标题
  - 第二行：公司 · 来源 · 发布时间
  - 第三行：链接按钮

交互细节：
- 点击标题或“打开链接”均可跳转外部新闻。
- 列表默认按发布时间倒序。

### 5. 空状态与错误态
- 空状态：当本次 output_count=0 时，提示“本次窗口内无新增消息”。
- 错误态：当最近一次失败时，在页面顶部显示紧凑错误条（包含失败原因）。
